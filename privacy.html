Privacy Policy

Last updated: June 2026 · Limkokwing University of Creative Technology, Sierra Leone · DLAW207

Plain-language summary: SierraLearn uses Supabase as its backend — a secure, GDPR-compliant cloud platform. Your account data (email, name, role) is stored in Supabase's PostgreSQL database hosted in the EU. Uploaded files are stored in Supabase Storage. We collect only what is necessary to run the platform. We do not sell, share, or use your data for advertising.

1. Who We Are

SierraLearn is an open educational resource platform developed as an academic project for the DLAW207 IT Law and IPR Legal Issues module at Limkokwing University of Creative Technology, Sierra Leone campus, Semester 06, March–July 2026. Its mission is to provide free, openly licensed educational materials to students and teachers across Sierra Leone, aligned with UN Sustainable Development Goal 4 — Quality Education.

The platform is licensed under the MIT License and qualifies as a Digital Public Good (DPG) under the DPG Standard.

2. What Data We Collect

We collect and store the following categories of data:

  • Account information: Your email address, full name, and role (student, teacher, or admin) when you register. This is stored in Supabase's secure PostgreSQL database.
  • Resource metadata: Titles, descriptions, subjects, levels, tags, license types, contributor names, upload timestamps, download counts, and aggregate rating data for resources you upload.
  • Uploaded files: If you upload a file (PDF, document, video, image), it is stored in Supabase Storage — a secure object storage service. Files are accessible via a public URL once a resource is approved.
  • Rating data: When you rate a resource, your user ID and star score (1–5) are stored to prevent duplicate ratings. Ratings are not publicly attributed to you by name.
  • Download counts: Anonymous numeric counters incremented each time a resource is downloaded. No user identity is linked to a download.
  • Session tokens: Supabase Authentication issues secure JWT session tokens stored in your browser to keep you logged in. These expire automatically.

We do NOT collect: payment information, national ID numbers, location data, browser history, biometric data, or any data beyond what is listed above.

3. How Data Is Stored and Protected

All platform data is stored in Supabase, a secure, open-source backend platform that provides:

  • PostgreSQL database with Row Level Security (RLS) — database-level access rules that ensure users can only access data they are permitted to see.
  • Supabase Authentication with industry-standard bcrypt password hashing and JWT session management.
  • Supabase Storage with access control policies — only approved resources are publicly downloadable; unapproved files are restricted.
  • Data encrypted at rest and in transit using TLS/SSL.
  • Supabase infrastructure is hosted on AWS in the EU region and is SOC 2 compliant.

4. Role-Based Access Control

SierraLearn enforces strict role-based access at both the application and database levels:

  • Guests (not logged in): Can browse and download approved resources only. Cannot upload, rate, or access any personal data.
  • Students: Can browse, search, filter, download, and rate resources. Cannot upload resources.
  • Teachers: Can do everything students can, plus submit resources for review. Submitted resources are invisible to the public until an admin approves them.
  • Admins: Have full access to the admin dashboard — approve or reject submissions, manage all users (including changing roles), view all platform statistics, and delete resources.

Role assignments are enforced by Supabase Row Level Security policies. No user can escalate their own role — only an admin can change another user's role.

5. Content Approval Process

All resources submitted by teachers are placed in a pending state and are not visible to any public user until reviewed and approved by an admin. This moderation layer ensures:

  • Only openly licensed, educationally appropriate content reaches the public library.
  • Copyright-infringing or inappropriate material can be rejected before publication.
  • Teachers receive a rejection reason when their submission is declined, enabling improvement and resubmission.

6. Open Licensing

All resources on SierraLearn must carry an approved open-source or open content license:

  • MIT License — permissive, maximum freedom, attribution appreciated.
  • Apache License 2.0 — permissive with explicit patent protection.
  • Creative Commons Attribution 4.0 (CC BY 4.0) — share and adapt with credit.
  • GNU General Public License 3.0 (GPL 3.0) — copyleft, derivatives must be open source.

By uploading a resource, contributors confirm they hold the rights to share the content under the chosen license. SierraLearn takes no ownership of any uploaded content.

7. How We Use Your Data

Your data is used solely to:

  • Authenticate you and maintain your session across visits.
  • Display your name as the contributor of resources you upload.
  • Enforce role-based access control across the platform.
  • Track download counts and aggregate ratings to surface popular resources.
  • Enable the admin to manage the platform and content moderation queue.
  • Export resource metadata as open JSON for data interoperability.

Your data is never used for advertising, profiling, analytics tracking, or shared with any third party beyond Supabase (our infrastructure provider).

8. Your Rights

  • Access: You can view your account information by signing in. Admins can export all resource metadata as JSON at any time.
  • Correction: Contact the platform administrator to correct your name or email.
  • Deletion: Admins can delete any user account and all associated resources. Users may request account deletion by contacting the admin.
  • Portability: All resource metadata is exportable as open JSON from the library page.
  • Withdrawal of consent: You may stop using the platform at any time. Session tokens expire automatically.

9. Data Protection Compliance

  • SierraLearn is designed in alignment with the Sierra Leone Data Protection Act and informed by GDPR principles of data minimisation, purpose limitation, and storage limitation.
  • Supabase, our infrastructure provider, is GDPR-compliant and SOC 2 certified.
  • Row Level Security ensures data isolation between users at the database level.
  • No data is processed for any purpose other than operating the platform.
  • SierraLearn qualifies as a Digital Public Good under the DPG Standard: open source, open data, no proprietary dependencies, SDG-aligned, and causes no harm.

10. Children's Privacy

SierraLearn serves students of all ages, including children. Registration requires an email address and name only — no sensitive personal data. No data about child users is shared with any third party. Parents or guardians may contact the admin to request deletion of a child's account.

11. Security Practices

  • Passwords are hashed using bcrypt — never stored in plain text.
  • All API calls use HTTPS with TLS encryption.
  • Supabase RLS policies enforce access rules at the database level, independent of application code.
  • All user-submitted text is sanitised before rendering to prevent Cross-Site Scripting (XSS).
  • JWT tokens are short-lived and refreshed automatically by Supabase Auth.
  • File uploads are restricted by type and size, and stored in a dedicated Supabase Storage bucket with access policies.

12. Third-Party Services

SierraLearn uses one third-party service:

  • Supabase (supabase.com) — database, authentication, and file storage. Supabase's privacy policy is available at supabase.com/privacy.

No analytics services, advertising networks, social media trackers, or other third-party scripts are used.
